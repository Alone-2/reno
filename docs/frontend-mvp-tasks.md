## 0. 执行纪律（强制）

1. **严格串行**：同一时间只做 **一项** `status=进行中` 的任务。
2. **完成才勾选**：实现 → **写/补测试** → **测试通过** → 更新本表「状态 / 完成日 / 验证命令」→ 才进入下一项。
3. **没完成的任务必须有 test 计划**；实现时 test 必须落地且 **绿** 才算完成。
4. **验证命令**（本机 `pnpm`/corepack 若坏掉，优先用下列）：

| 包             | 类型检查                                                                      | 测试                            |
| -------------- | ----------------------------------------------------------------------------- | ------------------------------- |
| `@reno/shared` | `cd packages/shared && ../../node_modules/.bin/tsc --noEmit -p tsconfig.json` | `cd packages/shared && vp test` |
| `@reno/web`    | `cd apps/web && ../../node_modules/.bin/tsc --noEmit -p tsconfig.json`        | `cd apps/web && vp test`        |

5. **金额**：全程 **分（integer）**，展示用 `fenToYuan`。
6. **UI**：Nest 暖巢 + shadcn **Base UI**；**新组件不加 Radix**。
7. **不顺带清后端旧 RBAC**（另开任务）；前端先 mock / 契约类型。
8. 每完成一项，把下表对应行改为 `✅ 完成`，并填写验证摘要。
